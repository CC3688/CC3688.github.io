# PropTypes 与 DefaultProps

import React, { Component } from 'react'
import PropTypes from 'prop-types'


class TodoItem extends Component {
    constructor(props) {
        super(props)
    }

    render() {
        const { content } = this.props;

        return (
            <div onClick={this.handleClick}>
                {content}
            </div>
        )
    }

    handleClick = () => {
        const { deleteItem, index} = this.props;
        deleteItem(index)
    }
    
}


TodoItem.propTypes = {            //这个propTypes p小写,它是要对这个组件,属性进行校验的意思
    content: PropTypes.string ,  //这个PropTypes是我们上面import的那个PropTypes,要和上面一样
    deleteItem:　PropTypes.func,
    index: PropTypes.number,
    test: PropTypes.string.isRequired, //字符串, 必传
    text: PropTypes.oneOfType([PropTypes.number, PropTypes.string]), //或者,是数字,或是字符串
}

TodoItem.defaultProps = {
    test: 'Hello world',  // 设置默认值
}


export default TodoItem

# props state render函数三者关系

当组件的state 或者 props 发生改变的时候, render函数就会重新执行

当父组件的render函数被运行时,它的子组件的render都将被重新运行

# 虚拟DOM

react 在数据与模版结合生成生成一个虚拟的DOM,

用虚拟DOM生成真实的DOM,显示在页面

虚拟DOM就是一个JS对象,用它来描述真实的DOM.

当数据state发生变化后,就会生成一个新的虚拟DOM

然后比较原始虚拟DOM 和 新的虚拟DOM的区别, 找到区别的地方

直接操作(有区别的)DOM, 改变(有区别的)DOM的内容

js创建虚拟DOM,即js对象,它性能消耗极低, js操作DOM性能消耗才大.

把对DOM的操作减少到极少, 提升了性能

在render 看到div 标签不要以为是真是的DOM, 实际上它是一个jsx的语法 render 是 jsx -> js对象 -> 真实的DOM

# ref

主要用来操作DOM

<input
    ref={(input)=> {this.input = input}}  // 这个函数的参数就可以这个dom, 名字随便起,
                                          //再把它赋值到 实例this 上面
/>
// this.input 指向的是这个 dom

然后就可以在方法里使用它

handleInputChange = ()=>{
    const value = this.input.value
}
  • 尽量不要用ref
  • 动画,不可以避免才需要使用
  • 数据更新后,需要获取DOM,请把获取DOM的操作放到setState的回调函数里执行,才能得到最新的
