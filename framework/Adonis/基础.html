# 基础

# create-react-app

可以全局安装, 也可以通过 npx 安装 (npx 下载 该包并使用该包, 确保每次都是最新版的)

> npx create-react-app my-app       //创建了一个my-app项目

创建完成后, 可以在要目录下 运行

npm run start 启动项目 start 命令可以省略 run

# 文件目录

|-- node_modules
|-- public
    |-- favico.ico      浏览器标签页的小图标
    |-- index.html      模板
    |-- manifest.json    
|-- src   项目的所有源代码都在这个目录下
    |-- App.js       组件
    |-- index.js     整个程序的入口文件       除了这两个文件其它的可以删除了
|-- .gitignore
|-- package.json
|-- yarn.lock 或 package-lock.json
|-- README.md

# react 组件

组件就是页面的各个部分

如: 头部组件 搜索框组件,它又可以分为输入组件和按纽组件

组件显示的内容, 由组件里的 render 返回的内容决定 (函数组件,则由函数返回值决定)

jsx 组件 开头, 必须以大写字母开头

# react 基础

一个类组件

import React, { Component } from 'react'
// import React, { Component, Fragment } from 'react'
// render 就可以直接 用 Fragment 不用  React.Fragment
export default class TodoList extends Component {

    constructor(props) {      //构造函数固定写法
        super(props)
        this.state = {       // 定义组件的状态
            inputValue: '',
            list: []
        }
        // 绑定this 如果是箭头函数就不用bind
        this.handleInputChange = this.handleInputChange.bind(this)
    }

    render() {
        return (
            <React.Fragment>
                <div>
                    // input显示的内容同 this.state.inputValue
                    // jsx 使用js 需要写在 {} 内
                    <input 
                        value={this.state.inputValue}
                        onChange={this.handleInputChange}
                     /> 
                    //绑定事件, 它和原生事件是有区别的,react是小驼峰, 原生是小写
                    <button>提交</button>
                </div>
                <ul>

                </ul>
            </React.Fragment>
        )
    }
    // 使用箭头函数, 绑定了this
    // 也可使用bind 即在 constructor 构造函数里绑定
    handleInputChange = (e)=>{
        e.target.value
        this.setState({
            inputValue: e.target.value   //只有通过setState 更新数据,视图才会更新
        })
    }
    

}
  • return 返回的jsx 必须有且只有一个 根原素
  • <React.Fragment> 是一个虚拟的元素,不生成任何元素,类似于 vue的template
